# Rough Idea: Comprehensive Project Documentation

Create comprehensive project documentation for EatMe v1, including diagrams of workflows. The documentation should be separated into files, each covering one area:

1. **Project Purpose** - Value for users, business goals, target audience
2. **Technical Stack** - All technologies, frameworks, and tools used
3. **CLI Commands** - All npm/pnpm commands used across the monorepo
4. **Web Portal Technical Documentation** - Architecture, routes, components, features
5. **Mobile App Technical Documentation** - Architecture, screens, navigation, services, state management
6. **Supabase Database Schema & Explanation** - All tables, relationships, RLS policies, RPC functions
7. **Supabase Edge Functions** - Each function's purpose, inputs, outputs, algorithms
8. **Application Workflows** - Every user and system workflow with Mermaid diagrams

All documentation should include Mermaid diagrams where appropriate (architecture, data flows, workflows, component relationships).
