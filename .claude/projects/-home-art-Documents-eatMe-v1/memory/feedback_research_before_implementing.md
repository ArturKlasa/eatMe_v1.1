Always do a thorough research of all affected code before implementing changes. Present findings and proposed solution, then wait for explicit approval before editing files.

**Why:** User wants to understand and approve the approach before any code is touched.

**How to apply:** For every issue — read all relevant files, trace the full data flow, present a clear summary of what will change and why, then ask for approval.
