## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser admin → edge function | Untrusted browser JS from an arbitrary origin attempts to read edge-function responses; the only gate on read access is the server-emitted CORS header set this helper produces. |
| runtime config → code | `ALLOWED_ORIGINS` (operator-set Supabase function secret) crosses into `buildCorsHeaders` via `Deno.env.get`; a missing/empty value must degrade safely (fail-closed), never widen access. |

## STRIDE Threat Register

ASVS L1, primary control V14.5 (CORS). Block-on: high. The mitigations below are implemented by `buildCorsHeaders` and proven by `cors.test.ts`.

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Information Disclosure | `buildCorsHeaders` ACAO value | mitigate | No wildcard `*` ever emitted; ACAO set ONLY on exact `allowlist.includes(origin)` match (D-03/D-08). Asserted by test case 1 (reflects) + the no-`'*'` grep gate. |
| T-02-02 | Spoofing / Information Disclosure | origin reflection path | mitigate | Origin is never reflected unconditionally; reflection is gated on exact-string allowlist membership — no regex/subdomain matcher (injection-surface-free per D-03). Asserted by test case 2 (disallowed → ACAO absent). |
| T-02-03 | Information Disclosure (fail-open) | `ALLOWED_ORIGINS` unset/misconfig | mitigate | Unset/empty env → empty allowlist → no ACAO (D-10 fail-closed), explicitly NOT fall-open to `*`. Asserted by test case 4. |
| T-02-04 | Tampering (cache poisoning) | shared/CDN cache keying | mitigate | `Vary: Origin` emitted unconditionally on every header set (D-11) so a cache cannot serve one origin's ACAO to another. Asserted by test cases 1/2/4 (Vary present). |
| T-02-05 | Elevation / token theft | credentialed CORS | accept (by design) | `Access-Control-Allow-Credentials` is deliberately omitted — admin auth is a bearer JWT in the `authorization` header, not cookies (RESEARCH Secondary Finding 1). Allow-Credentials + reflected origin would be a token-theft vector; absence removes it. Asserted by the no-credentials grep gate. |
