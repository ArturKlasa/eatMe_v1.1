## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| mobile client (anon or authed JWT) → PostgreSQL via PostgREST | Untrusted caller crosses into the DB; `auth.uid()` is resolved from the request JWT (NULL when anon). RLS is the ONLY enforcement point for per-user data isolation on these mobile-direct tables. |
| backend worker (service_role key) → PostgreSQL | Trusted backend; service_role BYPASSES RLS entirely. Explicit `ALL TO service_role` policies are codified for fidelity only (D-07), not for correctness. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
