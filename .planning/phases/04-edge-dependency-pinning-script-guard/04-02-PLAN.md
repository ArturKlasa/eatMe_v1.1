<guard_design_note>
Claude's-discretion API shape, resolved here (RESEARCH B.4 + PATTERNS):
- `parseGuard(argv: string[] = process.argv): { dryRun: boolean; apply: boolean; projectRef: string; limit: number }`
  - `apply = argv.includes('--apply')`
  - `dryRun = !apply` — so absent `--apply` the default is dry-run (D-09); `--dry-run` present or absent makes no difference to `dryRun` (it is an accepted no-op affirmation, never errors — D-10)
  - `projectRef` derived from `process.env.SUPABASE_URL` host: parse `https://<ref>.supabase.co` → `<ref>`; if `SUPABASE_URL` is empty/unparseable, return a safe sentinel like `"(unknown)"` (never throw — the script's own env-validation handles missing env). No `SUPABASE_PROJECT_REF` env exists anywhere; use `SUPABASE_URL` only (RESEARCH B.3).
  - `limit` parsed from a `--limit=N` arg and RETURNED (not consumed) so the dry-run → sample → full workflow (D-12) is preserved; scripts may use the guard's `limit` or keep their own parsing — either way the guard must not break `--limit` (Pitfall 5). `limit = 0` means "all".
- `announceTarget(g: { dryRun: boolean; projectRef: string }): void` — prints a loud banner BEFORE any write, on every run (D-11). The banner must make the LIVE-default inversion obvious, e.g. dry-run path prints `DRY RUN (no writes) — project <ref> — re-run with --apply to write`; apply path prints `⚠ APPLYING to project <ref>`. This generalizes the `apply-phase6-flag-fixes.ts:453-456` inline form verbatim.

Run convention: scripts are ts-node (`#!/usr/bin/env ts-node`), args via `process.argv` (Node, not Deno). The guard MUST parse `process.argv`, never Deno args. `import 'dotenv/config'` is run by each caller before reading env — the guard just reads `process.env.SUPABASE_URL`.
</guard_design_note>
