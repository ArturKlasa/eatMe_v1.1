<harness_constraints>
The harness MUST run under plain Node (v22.20 is available; no ts-node/tsx/jest/vitest in apps/mobile — confirmed). It must NOT import the live store (`index.ts`), because that pulls in `zustand` + `@react-native-async-storage/async-storage` + `@eatme/shared` (React Native / native deps that do not import under bare Node).

The serialization seam is exactly `JSON.stringify(currentState.permanent)` — its output is fully determined by the shape of the `permanent` object, whose defaults are `defaultPermanentFilters` (pure data, no native deps). So the harness compares:
  (a) the BASELINE serialized string = `JSON.stringify` of the pre-refactor `defaultPermanentFilters` (recovered from git: `git show <pre-08-01-commit>:apps/mobile/src/stores/filterStore.ts`, or simpler, an inline literal copy of the pre-refactor default object pasted into the harness as the frozen baseline), AND
  (b) the CURRENT serialized string = `JSON.stringify` of the post-refactor `defaultPermanentFilters` exported from `filterStore/defaults.ts`.

To read the current value without native deps: either (i) `node --experimental-strip-types` importing `defaults.ts` IF it has no native imports (defaults.ts imports only `../../utils/currencyConfig` → `@eatme/shared`; that chain MAY pull RN-incompatible code, so prefer option ii), OR (ii) the harness inlines a copy of the post-refactor `defaultPermanentFilters` object literal read directly from `defaults.ts` and asserts it equals the frozen pre-refactor baseline literal. Option (ii) is the robust, dependency-free path: paste both object literals (pre and post) into the `.mjs` and assert `JSON.stringify(pre) === JSON.stringify(post)` for the defaults AND for a populated sample (e.g. dietPreference 'vegan', exclude.noMeat true, defaultNutrition.enabled true, cuisinePreferences ['italian']). Document which option was used in the SUMMARY.
</harness_constraints>
