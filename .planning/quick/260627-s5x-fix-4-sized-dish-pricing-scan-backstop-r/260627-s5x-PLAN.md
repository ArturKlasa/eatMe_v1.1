# Quick Task 260627-s5x — Plan

Operator issue #4: a dish whose price lives entirely in size variants (e.g.
"Pizza — Chica $90 / Mediana $120 / Grande $150", no standalone base) shows the
card price as `—` and the size prices as `+$90` deltas instead of final prices.

## Root cause (code-confirmed)
- Scan prompt: `price: null if not shown` → `dish.price = null`; per-option
  printed prices captured as `price_delta` (override reserved for non-linear
  qty). `display_price_prefix` defaults to `'exact'` (never auto-derived).
- Display is correct per-field (`= override` vs `+ delta`,
  `DishRowEditor.tsx:631-640`); `formatPriceCell(null)` → `—`
  (`DishRowEditor.tsx:67`, mobile `DishMenuItem.tsx:53`). The data is the bug.

## Approach (user-chosen: BOTH)
Scan-time backstop fixes NEW scans at the data layer; display fallback rescues
already-confirmed live dishes without a re-scan or migration.

## Tasks

### Task 1 — Shared helper (`packages/shared`)
- Add `deriveSizeFromPrice(price, groups)` to `@eatme/shared`
  (`packages/shared/src/logic/`), exported from the package index.
- Returns a number (the "from" price) ONLY when: `price == null` AND there is
  exactly ONE qualifying group — `selection_type === 'single'`,
  `min_selections >= 1`, `>= 2` options, every option carries a positive
  effective price (`price_override ?? price_delta > 0`). Returns the MIN
  effective price; otherwise `null`. Conservative: a based-price dish with a
  `+$3` upgrade (price set) and optional add-ons (min 0) never qualify.
- Unit tests (vitest) in `packages/shared`: sized delta dish → min; based+upgrade
  → null; optional add-on group → null; multiple qualifying groups → null;
  override-priced size group with null base → min.

### Task 2 — Worker backstop (Deno, `menu-scan-worker/index.ts` + `test.ts`)
- New `deriveSizeGroupPricing(dishes)` run in `runExtraction` after
  `normalizeModifierPriceOverrides`. For a `price == null` dish with exactly one
  qualifying single-select required group whose options are all
  `price_override == null && price_delta > 0` (>=2): set each option's
  `price_override = price_delta`, `price_delta = 0`; set `dish.price = min`;
  set `dish.display_price_prefix = 'from'`.
- Add an optional `display_price_prefix` to the scanned-dish output type so the
  worker can emit it (model schema unchanged; `reviewHelpers.ts:101` already
  reads `d.display_price_prefix ?? 'exact'`).
- Tests: sized delta dish → override + price=min + prefix 'from'; protein
  upgrade on a based dish → untouched; existing override "12 wings $45" group →
  untouched; mixed delta-with-a-zero group → untouched (skip).

### Task 3 — Mobile display fallback (`DishMenuItem.tsx`)
- Import `deriveSizeFromPrice`. When it returns a value, render
  `t('restaurant.price.from', { price: formatPrice(min, currency) })` on the
  card. New backstopped dishes (price set + prefix 'from') still hit the normal
  path, so no double-handling. Verified on-device by the user.

### Task 4 — Admin display fallback (`DishRowEditor.tsx`)
- Same helper in the row price cell (~L265): show `from <price>` instead of `—`
  for existing null-price sized dishes (derive groups from the dish's
  `option_groups`). Published-dish editor only; the scan-review screen gets
  correct data from the worker backstop.

## must_haves
- truths:
  - A NEW scan of a size-priced dish stores `dish.price = cheapest size`,
    `display_price_prefix = 'from'`, and the size options as `price_override`
    (final prices), not deltas.
  - The backstop never alters a based-price dish, an optional add-on group, an
    existing override-quantity group, or a dish with >1 qualifying group.
  - An EXISTING null-price size-priced dish renders `from $<min>` on the mobile
    card and the admin row (instead of `—`).
- artifacts:
  - `packages/shared/src/logic/*` (deriveSizeFromPrice + index export) + tests
