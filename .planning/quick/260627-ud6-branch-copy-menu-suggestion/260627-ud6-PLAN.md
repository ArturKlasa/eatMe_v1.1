# Quick Task 260627-ud6 — Plan

Enhance the **existing** copy-menu feature (issue #16) so it is **proactive**.
Today `CopyMenuSection` renders on a restaurant's detail page when it has no
menus and makes the operator blind-search for a source. This task auto-ranks
likely branches by trigram name similarity and offers one-tap copy, with the
manual search as fallback. Copy mechanics (RPC `admin_copy_restaurant_menu`)
and placement are unchanged.

Locked decisions (see CONTEXT.md): match **all restaurants** by name similarity;
eligible source = **any restaurant with ≥1 dish**; UX = **top 3 ranked,
dismissible** cards above the manual search; silent fallback (no empty state)
when nothing clears the threshold.

## Task 1 — DB: `suggest_copy_source_restaurants` RPC (migration 179)

**Files:** `infra/supabase/migrations/179_suggest_copy_source_restaurants.sql`
(+ `179_REVERSE_ONLY_suggest_copy_source_restaurants.sql`)

**Action:** New function returning the top trigram name matches for a target
restaurant, mirroring the `CopySourceCandidate` columns.

```sql
CREATE OR REPLACE FUNCTION public.suggest_copy_source_restaurants(
  p_target_restaurant_id uuid,
  p_limit int DEFAULT 3
)
RETURNS TABLE (
  id uuid, name text, city text, status text, dish_count bigint
)
LANGUAGE sql
STABLE
SECURITY INVOKER
SET search_path = public, extensions
AS $$
  SELECT r.id, r.name, r.city, r.status::text,
         (SELECT count(*) FROM public.dishes d WHERE d.restaurant_id = r.id) AS dish_count
  FROM public.restaurants r
  CROSS JOIN LATERAL (
    SELECT name FROM public.restaurants WHERE id = p_target_restaurant_id
  ) tgt
  WHERE r.id <> p_target_restaurant_id
    AND similarity(public.f_unaccent(lower(r.name)), public.f_unaccent(lower(tgt.name))) >= 0.30
    AND EXISTS (SELECT 1 FROM public.dishes d WHERE d.restaurant_id = r.id)
  ORDER BY similarity(public.f_unaccent(lower(r.name)), public.f_unaccent(lower(tgt.name))) DESC, r.name
  LIMIT p_limit;
$$;

REVOKE ALL ON FUNCTION public.suggest_copy_source_restaurants(uuid, int) FROM PUBLIC;
GRANT EXECUTE ON FUNCTION public.suggest_copy_source_restaurants(uuid, int) TO authenticated;
GRANT EXECUTE ON FUNCTION public.suggest_copy_source_restaurants(uuid, int) TO service_role;
```

REVERSE_ONLY drops the function.

**Notes:** Follows the migration 126 precedent — explicit `similarity() >= cutoff`
(NOT the `%` operator, which forces pg_trgm's default 0.3 GUC and would make any
other floor dead code), so the threshold is deterministic regardless of session
`set_limit`. `f_unaccent(lower(...))` (migration 126) folds case + Spanish
accents so "Café X" matches "Cafe X". `0.30` is recall-favoring but bounded by
the top-3 cap + dismiss — a one-line tune. The explicit form can't use the GIN
trgm index, but `restaurants` is operator-scale (seq scan negligible). `STABLE`;
called via the service-role client (RLS bypassed) like the copy RPC, so
`SECURITY INVOKER` suffices.

**Verify:** SQL parses; column list matches the action's mapper. (Migration
applied out-of-band by operator — REST-only locally.)

**Done:** migration + reverse file written, numbered 179, header comment links
to issue #1 and migration 116c/160.

## Task 2 — Action: `suggestCopySourceRestaurants`

**Files:** `apps/admin/src/app/(admin)/restaurants/[id]/actions/copyMenu.ts`

**Action:** Add a `withAdminAuth` server action that calls the new RPC via the
service client and maps rows to the existing `CopySourceCandidate[]`:

```ts
type SuggestRpc = {
  rpc: (name: 'suggest_copy_source_restaurants',
        args: { p_target_restaurant_id: string; p_limit?: number })
    => Promise<{ data: Array<{ id: string; name: string; city: string | null;
        status: string; dish_count: number }> | null;
        error: { message?: string } | null }>;
};

export const suggestCopySourceRestaurants = withAdminAuth(
  async (_ctx, targetRestaurantId: string): Promise<ActionResult<CopySourceCandidate[]>> => {
    const service = createAdminServiceClient();
    const { data, error } = await (service as unknown as SuggestRpc)
      .rpc('suggest_copy_source_restaurants', { p_target_restaurant_id: targetRestaurantId, p_limit: 3 });
    if (error) return { ok: false, formError: error.message };
    return { ok: true, data: (data ?? []).map(r => ({
      id: r.id, name: r.name, city: r.city, status: r.status, dish_count: r.dish_count })) };
  }
);
```

**Verify:** `cd apps/admin && npx tsc --noEmit` clean.

**Done:** action exported; reuses `CopySourceCandidate`; no change to existing
`searchCopySourceRestaurants` / `adminCopyRestaurantMenu`.
